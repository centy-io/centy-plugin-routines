# Extract catalog to markdown files

## Description

Extract the routine catalog from `src/catalog.ts` into separate markdown files.

Each routine's prompt/instructions should live in its own `.md` file rather than being inline in TypeScript. This improves readability, makes it easier to edit prompts, and enables better diffs when routine content changes.

## Acceptance Criteria

- Each routine's content is extracted to a dedicated markdown file
- `src/catalog.ts` references the markdown files instead of containing inline content
- All existing functionality remains unchanged
